# AI and MCP Usage Policy

This repository allows AI-assisted development through tools such as Cursor and MCP servers, subject to the following boundaries.

## Core rule

AI/MCP tools may read, explain, generate, and diagnose repository content.

AI/MCP tools must not be used as the direct execution boundary for production infrastructure changes.

GitHub Actions, required reviews, protected branches, GitHub Environments, Terraform plan review, and Terraform/HCP Terraform approval workflows remain the source of enforcement.

## Approved uses

AI/MCP tools may be used to:

* Inspect source code, documentation, workflow logs, and Terraform configuration.
* Generate proposed code, workflow, or Terraform changes.
* Explain GitHub Actions failures.
* Look up current provider, module, and API documentation.
* Draft pull request descriptions and remediation notes.

## Restricted uses

AI/MCP tools must not be used to:

* Apply Terraform changes directly to production.
* Bypass pull requests, branch protection, or required reviews.
* Disable security checks, required status checks, or approval gates without explicit maintainer review.
* Use broad write tokens as the default development configuration.
* Store secrets, cloud credentials, GitHub tokens, or HCP Terraform tokens in committed MCP configuration.

## GitHub MCP policy

The default GitHub MCP configuration should be read-only.

Write-capable GitHub MCP access may only be used intentionally for scoped tasks such as creating a pull request or dispatching a workflow, and should not be left enabled as the default configuration.

## Terraform MCP policy

Terraform MCP should be used for documentation-backed assistance, provider/module lookup, and review support.

Terraform MCP output must be reviewed before being committed.

Terraform plans must be generated by approved CI/CD or Terraform/HCP Terraform workflows.

Terraform applies must occur only through approved GitHub Actions environments or Terraform/HCP Terraform runs with the required approvals.

## Pull request expectations

Pull requests that include AI/MCP-assisted changes must confirm that:

* The author reviewed the generated changes.
* Terraform plans or CI results were reviewed where applicable.
* No secrets or local MCP configuration files were committed.
* Production-impacting changes remain behind the normal approval process.
